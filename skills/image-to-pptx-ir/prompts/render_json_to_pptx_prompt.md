# Prompt: 根据 Render JSON 生成可编辑 PPTX

你是一名 PPTX 高保真还原工程师。

我会提供 render.json / PPTX-IR。请根据它生成可编辑 PPTX。

## 要求

1. 严格按 render.json 绘制，不要自由设计。
2. 坐标使用原图像素坐标映射到 16:9 PPT。
3. 文本必须是可编辑文本框。
4. 矩形、圆角矩形、线条、箭头必须是 PPT 原生元素。
5. 简单图标使用精确匹配的 SVG 或图标库资源；复杂 AI 图标严格使用 render.json 绑定的透明高清素材，不得重新绘制或替换。
6. 对于 `wrap: false` 的文本，必须关闭自动换行，并设置 shrink/fit。
7. 对于 `dash: dash` 的线条，必须映射成 PPT 支持的虚线样式。
8. 对于 `mustNotBeSegmented: true` 的 connector，必须生成一根完整线条。
9. 生成后执行：
   - 元素越界检查
   - 重叠检查
   - 导出 PNG 预览
   - 与源图逐项核对
   - 验证复杂图标真实 Alpha、透明边缘和 sourceBBox/alphaBBox 对齐
   - 把复杂图标回贴到源图位置后执行局部相似度复核

## 重点检查项

- 左右大框是否为直角矩形
- 内部虚线框是否正确
- 所有箭头实线/虚线/方向是否正确
- 黄色加载配置箭头是否是一侧一根完整虚线箭头
- 文本是否被 PowerPoint 自动换行
- 子元素是否超出父容器
- 复杂图标是否被拉伸、错位、缩小或出现白边/灰边
- 复杂图标素材是否误包含文字、箭头、虚线或卡片边框
